Сущности и модели данных (Domain Entities)

=== Аутентификация и Авторизация (Identity) ===

Account (Сущность аутентификации — отдельно от профиля):
  Id (PK), Email, PasswordHash (nullable — при OAuth может отсутствовать),
  EmailConfirmed, EmailConfirmationToken (nullable),
  PasswordResetToken (nullable), PasswordResetTokenExpiry (nullable),
  GoogleId (для OAuth 2.0 привязки),
  RefreshToken, RefreshTokenExpiryTime,
  Role (FK → Role), IsActive, DateCreated, LastLoginAt.

Role: Система ролей (User, Admin). Управление доступом к контенту и модерации.

=== Профиль пользователя (отдельная сущность) ===

UserProfile (Личные данные пользователя, связан 1:1 с Account):
  Id (PK), AccountId (FK → Account),
  FirstName, LastName,
  Username (уникальный, отображаемый ник),
  Bio, AvatarUrl,
  DateOfBirth,
  Address (город / страна / регион — текстовое поле или структура),
  CreatedAt, UpdatedAt.

=== Ядро контента (Media Items) ===

MediaItem (Базовый абстрактный класс/сущность):
  Id, Title, OriginalTitle, ReleaseYear,
  CoverImageUrl, Description,
  GlobalRating (средняя оценка всех пользователей),
  ExternalId (идентификатор из внешнего API — например, TMDB ID, Steam AppID, ISBN и т.д.),
  ExternalSource (источник данных: TMDB, GoogleBooks, Steam, Spotify),
  CreatedAt, UpdatedAt.

  Важно: MediaItem — это общий пул контента. Когда пользователь добавляет контент,
  система сначала ищет его в БД по ExternalId + ExternalSource. Если найден — используется
  существующий. Если нет — создаётся новый из данных внешнего API. Таким образом,
  один и тот же фильм/книга/игра/трек существует в БД в единственном экземпляре.

Типы контента (Наследники или связанные сущности):

  Movie:
    Director (режиссёр), Duration (длительность в минутах),
    Cast (актёрский состав — строка или коллекция),
    Country (страна производства), Language (язык оригинала),
    Genre (жанр), Budget (бюджет, nullable),
    TrailerUrl (ссылка на трейлер, nullable).

  Book:
    Author (автор), PageCount (количество страниц),
    Genre (жанр), Publisher (издательство),
    ISBN (международный номер книги),
    Language (язык оригинала),
    Series (серия/цикл, nullable), SeriesOrder (порядковый номер в серии, nullable).

  Game:
    Developer (разработчик), Publisher (издатель),
    Platform (платформы — строка или коллекция),
    Genre (жанр), AveragePlayTime (среднее время прохождения в часах),
    AgeRating (возрастной рейтинг, например PEGI/ESRB),
    Engine (игровой движок, nullable),
    IsMultiplayer (поддержка мультиплеера).

  Music:
    Artist (исполнитель), Album (альбом),
    Genre (жанр), Duration (длительность трека в секундах),
    Label (лейбл/издатель),
    TrackNumber (номер трека в альбоме, nullable),
    Language (язык), ReleaseType (Single / Album / EP).

=== Сердце проекта — Таймлайн (Timeline Entry) ===

Entry: Это «Карточка», связывающая Пользователя и Контент.

  UserId (FK → UserProfile) и MediaItemId (FK → MediaItem).

  Status: Текущее состояние (например: Planned, InProgress, Completed, Dropped, OnHold).

  Rating: Личная оценка пользователя от 0 до 10.

  Review: Текстовое описание/мнение (Markdown support).

  TimeSpent: Потраченное время (в минутах или часах).

  IsPrivate: Флаг, скрывать ли эту запись из публичного таймлайна.

  IsFavorite: Флаг, является ли карточка избранной.

  CreatedAt / UpdatedAt: Системные метки времени.

EntryEvent (История изменений статуса — дочерняя сущность Entry):
  Id (PK), EntryId (FK → Entry),
  Status (новый статус: Planned, InProgress, Completed, Dropped, OnHold),
  DateTime (когда произошла смена статуса),
  Note (комментарий пользователя к событию, nullable).

  Логика: Каждый раз, когда пользователь меняет статус Entry, создаётся новая запись
  EntryEvent. Поле Status в самом Entry всегда отражает последний актуальный статус.
  EntryEvent хранит полную хронологию: "Начал → Поставил на паузу → Возобновил → Завершил".
  Именно эти события формируют детальный таймлайн взаимодействия с контентом.

=== Социальное взаимодействие (Social) ===

Follow (Подписка на пользователя):
  Id (PK), FollowerId (FK → UserProfile — кто подписался),
  FollowingId (FK → UserProfile — на кого подписался),
  CreatedAt.
  Ограничение: уникальная пара (FollowerId, FollowingId). Нельзя подписаться на себя.

Like (Реакция на запись другого пользователя):
  Id (PK), UserId (FK → UserProfile — кто лайкнул),
  EntryId (FK → Entry — какую запись лайкнул),
  CreatedAt.
  Ограничение: уникальная пара (UserId, EntryId). Один лайк на одну запись.

Comment (Комментарий к записи другого пользователя):
  Id (PK), UserId (FK → UserProfile — кто написал),
  EntryId (FK → Entry — к какой записи),
  Text (текст комментария),
  CreatedAt, UpdatedAt.

=== Уведомления (Notifications) ===

Notification (Уведомление пользователю или админу):
  Id (PK), RecipientId (FK → UserProfile — кому адресовано),
  ActorId (FK → UserProfile — кто вызвал действие, nullable для системных уведомлений),
  Type (NotificationType: NewFollower, EntryLiked, NewComment, AdminAlert, System),
  EntityType (строка — тип затронутой сущности: "Entry", "Comment", "Follow" и т.д.),
  EntityId (Guid — ID затронутой сущности, nullable),
  Message (текст уведомления),
  IsRead (bool — прочитано ли),
  CreatedAt.

Примеры уведомлений:
  Юзеру: «@maria подписалась на вас», «@alex лайкнул вашу запись Interstellar»,
          «@john оставил комментарий к вашей записи».
  Админу: «Пользователь @alex удалил запись #456», «Новый пользователь зарегистрирован».
  Системные: «Добро пожаловать в TLOM!», «Заполните свой профиль».

Доставка: сохраняются в БД + push через SignalR (real-time).

=== Аудит-лог (для админ-панели) ===

AuditLog (Запись аудита действий пользователей):
  Id (PK), AccountId (FK → Account — кто совершил действие),
  Action (тип действия: Created, Updated, Deleted),
  EntityType (тип сущности: Entry, MediaItem и т.д.),
  EntityId (ID затронутой сущности),
  OldValues (JSON-снимок до изменения, nullable),
  NewValues (JSON-снимок после изменения, nullable),
  IpAddress, UserAgent,
  Timestamp (дата и время действия).
