Архитектура страниц фронтенда (Routing & Pages)

В этом документе описана структура маршрутизации (React Router v6) и функционал каждой отдельной страницы (Page Components) для фронтенда платформы Time Line Of Me.

=== 1. Публичные страницы (Public Routes) ===
Доступны без авторизации.

1.1. Landing Page (/)
— Описание: Визитная карточка проекта.
— UI/Функционал: 
  • Полноэкранная scroll-driven анимация пути человека через медиа (от детства к зрелости) с использованием Framer Motion.
  • Демо-версия таймлайна, появляющаяся при скролле.
  • Кнопки призыва к действию: "Начни свою историю", ведущие на регистрацию.
— Данные: Статическая страница.

1.2. Auth Pages (/login, /register)
— Описание: Страницы входа и регистрации.
— UI/Функционал:
  • Формы (email/пароль) с валидацией через React Hook Form + Zod.
  • Большая кнопка "Продолжить с Google" (OAuth 2.0).
  • Обработка состояний загрузки и ошибок (неверный пароль и т.д.).
— Данные: Отправка мутаций (TanStack Query) к эндпоинтам аутентификации, сохранение JWT-токена, обновление глобального стейта (Zustand).

=== 2. Приватные страницы (Private Routes) ===
Доступны только после авторизации. Обернуты в ProtectedRoute компонент.

2.1. Onboarding / Profile Setup (/onboarding)
— Описание: Страница, на которую юзер попадает при самом первом входе после регистрации.
— UI/Функционал:
  • Пошаговый мастер (Stepper): Загрузка аватарки, выбор уникального Username, написание Bio, указание даты рождения.
— Данные: Мутация загрузки файла (в AWS S3) и обновление UserProfile.

2.2. Dashboard / Home (/dashboard)
— Описание: Главная стартовая панель пользователя.
— UI/Функционал:
  • Верхний блок: Краткая визуальная статистика (сколько книг/фильмов/игр добавлено).
  • Средний блок: "Последние действия" пользователя (быстрый доступ к карточкам InProgress).
  • Правая колонка/Блок ниже: Мини-версия ленты подписок (Feed).
  • Плаваящая кнопка (FAB): "+ Добавить контент" для быстрого перехода к поиску.
— Данные: Параллельные запросы (useQueries) для статистики, последних Entries и краткого фида.

2.3. Feed Page (/feed)
— Описание: Лента активности подписок.
— UI/Функционал:
  • Вертикальная лента с карточками событий (пользователь X начал игру Y, пользователь Z поставил 10/10 фильму).
  • Возможность ставить Лайки и оставлять Комментарии прямо из ленты.
  • Infinite Scroll (бесконечная прокрутка) через useInfiniteQuery.

2.4. My Timeline (/timeline)
— Описание: Основной визуальный продукт проекта.
— UI/Функционал:
  • Визуальная лента времени с карточками контента.
  • Панель фильтров: Фильтрация по типам, статусам (Completed, InProgress) и годам.
  • Сортировка (сначала новые/старые, по рейтингу).
  • Infinite Scroll.

2.5. Search & Add Content (/search)
— Описание: Единая точка входа для добавления контента из внешних баз.
— UI/Функционал:
  • Большой инпут поиска с Debounce.
  • Tabs (Вкладки) для переключения: Фильмы (TMDB) / Книги (Google Books) / Игры (Steam/CheapShark) / Музыка (Spotify).
  • Сетка результатов. Кнопка "Добавить в таймлайн".
— Данные: Вызов внешних API через бэкенд, затем мутация CreateEntry.

2.6. Entry Details & Edit (/entry/:id)
— Описание: Детальная страница конкретной карточки (Entry) в таймлайне пользователя.
— UI/Функционал:
  • Большой постер/обложка контента (MediaItem).
  • Управление статусом: Выпадающий список (Planned → InProgress → Completed).
  • История статусов (EntryEvent): Визуальный лог того, как менялся статус (когда начал, когда закончил).
  • Поля ввода: Оценка (1-10 звезд), Ревью (Rich Text Editor для Markdown), Потраченное время, Флаги "Private" и "Favorite".
— Данные: Форма с автосохранением или кнопкой "Сохранить" (React Hook Form). Мутации на обновление статуса или данных ревью.

2.7. Analytics / Stats (/stats)
— Описание: Личная статистика (Spotify Wrapped-style).
— UI/Функционал:
  • Красивые графики (Recharts или Chart.js).
  • Выбор года или "За всё время".
  • Блоки: "Любимый жанр", "Топ оценки", "Распределение времени".
  • Кнопка "Поделиться" (генерация красивой картинки для соцсетей или шаринг ссылки).

=== 3. Публичные профили пользователей (Public User Routes) ===

3.1. Public Profile (/@:username)
— Описание: Страница чужого пользователя (если она не скрыта настройками приватности).
— UI/Функционал:
  • Шапка: Аватарка, Bio, счётчики контента, кнопка "Подписаться / Отписаться".
  • Публичный таймлайн этого пользователя (видны только IsPrivate = false записи).
  • Блок с его любимым контентом (Favorites).
— Данные: Запрос профиля по юзернейму, запрос публичного таймлайна.

=== 4. Административная панель (Admin Routes) ===
Доступны только для Role = Admin. Отдельный Layout без лишних украшательств.

4.1. Admin Dashboard (/admin)
— Описание: Общая сводка.
— UI/Функционал: Счётчики активных пользователей, контента в БД, нагрузки (опционально).

4.2. Audit Logs (/admin/audit)
— Описание: Таблица аудита системы.
— UI/Функционал:
  • Таблица (Shadcn Data Table) с колонками: Дата, Юзер, Действие, Сущность.
  • Модальное окно для просмотра OldValues и NewValues (сравнение JSON).
  • Сложная фильтрация. Пагинация со страницами (Offset-based).

4.3. Users Management (/admin/users)
— Описание: Управление пользователями.
— UI/Функционал:
  • Таблица пользователей.
  • Возможность заблокировать/разблокировать аккаунт.
  • Кнопка "Посмотреть контент от лица юзера" или принудительное редактирование/удаление запрещенного контента.

=== Структура проекта (Feature-Sliced Design пример) ===
src/
├── app/                  # Инициализация React, глобальные стили, роутер (App.tsx)
├── pages/                # Сборка страниц (компонуют виджеты/фичи)
│   ├── public/           # Landing, Login, Register
│   ├── private/          # Dashboard, Timeline, Entry, Stats
│   ├── user/             # PublicProfile
│   └── admin/            # AdminDashboard, AuditLogs
├── widgets/              # Самостоятельные крупные блоки (Header, Sidebar, UserFeed, TimelineView)
├── features/             # Бизнес-логика конкретных действий (AuthForm, CreateEntry, FollowUser, ChangeEntryStatus)
├── entities/             # Сущности (MediaItemCard, UserProfileHeader - dumb компоненты)
└── shared/               # UI-кит (Shadcn компоненты), API-клиент (axios/fetch config), утилиты, константы

=== Навигация и Layouts ===
— PublicLayout: Без сайдбара, простой прозрачный хедер (для лендинга).
— MainLayout: Боковой сайдбар (навигация: Главная, Таймлайн, Лента, Статистика) + Верхний хедер с поиском, меню профиля, переключателем тем (Dark/Light) и переключателем языков (az/ru/en).
— AdminLayout: Отдельная строгая панель навигации (Dashboard, Аудит, Пользователи).

=== UI Фичи и Глобальный Стейт ===
— Theming: Глобальный ThemeProvider, управляющий классом `.dark` в документе.
— Локализация (i18n): Использование `react-i18next` с файлами локализации (en.json, ru.json, az.json) и LanguageDetector.
