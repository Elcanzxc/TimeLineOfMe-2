# Документация по API Endpoints (TLOM Backend)

Ниже представлен полный список эндпоинтов, которые были реализованы в бэкенде. Frontend будет обращаться именно к ним. 
Базовый URL (при локальной разработке): `https://localhost:7068` (или `http://localhost:5170`).

---

## 1. Auth (Аутентификация и Авторизация)
Отвечает за регистрацию, вход и выдачу JWT токенов.

* **POST** `/api/auth/register`
  * **Тело:** `{ "email": "...", "password": "...", "username": "..." }`
  * **Описание:** Регистрация нового пользователя. Отправляет письмо с подтверждением.
* **POST** `/api/auth/login`
  * **Тело:** `{ "email": "...", "password": "..." }`
  * **Описание:** Вход по email и паролю. Возвращает `AccessToken` и `RefreshToken`. (Требует подтвержденный Email).
* **POST** `/api/auth/refresh`
  * **Тело:** `{ "refreshToken": "..." }`
  * **Описание:** Обновление протухшего `AccessToken`.
* **POST** `/api/auth/google`
  * **Тело:** `{ "idToken": "..." }` (токен, полученный от Google на клиенте)
  * **Описание:** Вход или автоматическая регистрация через Google OAuth. Почта считается автоматически подтвержденной.
* **GET** `/api/auth/confirm-email?userId={id}&token={token}`
  * **Описание:** Подтверждение почты после регистрации.
* **POST** `/api/auth/forgot-password`
  * **Тело:** `{ "email": "..." }`
  * **Описание:** Запрос на сброс пароля. Отправляет письмо со ссылкой.
* **POST** `/api/auth/reset-password`
  * **Тело:** `{ "email": "...", "token": "...", "newPassword": "..." }`
  * **Описание:** Установка нового пароля с использованием токена из письма.

## 2. MediaItems (Контент: Фильмы, Книги, Игры, Музыка)
Интеграция с внешними API (TMDB, Google Books, Steam/CheapShark, Spotify).

* **GET** `/api/mediaitems/search?query={текст}&mediaType={Movie|Book|Game|Music}`
  * **Описание:** Ищет контент во внешних источниках и возвращает список. Флаг `IsImported` показывает, есть ли этот контент уже в нашей БД.
* **POST** `/api/mediaitems/import`
  * **Тело:** `{ "externalId": "...", "source": "TMDB|GoogleBooks|Steam|Spotify", "mediaType": "..." }`
  * **Описание:** Загружает полные данные из внешнего API и сохраняет в локальную БД. Возвращает локальный `Id` созданного `MediaItem`.
* **GET** `/api/mediaitems/{id}`
  * **Описание:** Получить информацию о контенте из нашей локальной базы.

## 3. Entries (Записи в таймлайне)
Главный функционал трекинга.

* **POST** `/api/entries`
  * **Тело:** `{ "mediaItemId": "...", "status": "Planned|InProgress|Completed|Dropped", "isPrivate": false, "rating": 5, "review": "..." }`
  * **Описание:** Добавить запись в свой таймлайн. `MediaItemId` берется из результата импорта.
* **GET** `/api/entries`
  * **Описание:** Глобальная лента (Feed). Возвращает записи текущего пользователя, его подписок (друзей) и публичные записи. Поддерживает Cursor Pagination (`?cursor=...`).
* **GET** `/api/entries/{id}`
  * **Описание:** Получить конкретную запись по её ID.
* **PUT** `/api/entries/{id}`
  * **Тело:** `{ "status": "...", "rating": ..., "review": "...", "isPrivate": ... }`
  * **Описание:** Обновить свою запись (изменить статус, поставить оценку и т.д.).
* **DELETE** `/api/entries/{id}`
  * **Описание:** Удалить свою запись.
* **GET** `/api/users/{userId}/entries`
  * **Описание:** Получить таймлайн конкретного пользователя (учитывая настройки приватности).

## 4. Social (Социальное взаимодействие)
Подписки, поиск пользователей.

* **GET** `/api/users/search?query={текст}`
  * **Описание:** Поиск пользователей по имени (Username).
* **GET** `/api/users/{id}`
  * **Описание:** Профиль пользователя (статистика, аватарка, био).
* **POST** `/api/users/onboarding`
  * **Тело:** `{ "username": "...", "firstName": "...", "bio": "...", "dateOfBirth": "..." }`
  * **Описание:** Завершение регистрации (первоначальная настройка профиля).
* **PUT** `/api/users/profile`
  * **Тело:** `{ "firstName": "...", "bio": "...", "city": "...", "isProfilePrivate": ... }`
  * **Описание:** Обновление данных профиля текущего пользователя.
* **POST** `/api/users/{id}/follow`
  * **Описание:** Подписаться на пользователя.
* **DELETE** `/api/users/{id}/follow`
  * **Описание:** Отписаться от пользователя.
* **GET** `/api/users/{id}/followers`
  * **Описание:** Список подписчиков пользователя.
* **GET** `/api/users/{id}/following`
  * **Описание:** Список подписок пользователя (на кого он подписан).

## 5. Stats (Статистика)
Аналитика профиля.

* **GET** `/api/users/{userId}/stats`
  * **Описание:** Возвращает агрегированную статистику пользователя:
    * Сколько всего фильмов просмотрено (в часах/минутах)
    * Сколько книг прочитано (по страницам)
    * Сколько пройдено игр
    * Любимые жанры

---

## 6. Notifications (Уведомления)
Работают в реальном времени через SignalR, но также имеют REST API для управления.

* **GET** `/api/notifications`
  * **Описание:** Получить историю своих уведомлений (пагинация).
* **PUT** `/api/notifications/{id}/read`
  * **Описание:** Отметить конкретное уведомление как прочитанное.
* **PUT** `/api/notifications/read-all`
  * **Описание:** Отметить все свои уведомления как прочитанные.

**SignalR Hub:** `/hubs/notifications`
* Подключение требует AccessToken. На фронтенде нужно подписываться на событие `ReceiveNotification`.

## 7. Admin (Админка)
Доступно только пользователям с ролью `Admin`.

* **GET** `/api/admin/audit-logs`
  * **Описание:** Просмотр полного аудита системы (кто, что и когда изменил). Offset пагинация.
* **GET** `/api/admin/users`
  * **Описание:** Список всех пользователей системы (для блокировки/управления).
* **GET** `/api/admin/stats`
  * **Описание:** Глобальная статистика платформы (для админ-дашборда).
* **PUT** `/api/admin/users/{id}/roles`
  * **Тело:** `{ "roles": ["Admin", "User"] }`
  * **Описание:** Назначить/забрать роли у пользователя.
