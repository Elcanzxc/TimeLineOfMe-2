Название проекта: Time Line Of Me (TLOM)

Суть продукта (Концепция): > TLOM — это B2C веб-платформа, представляющая собой живую хронологию жизни пользователя через призму искусства и медиа, которые он потребляет. Это единый визуальный таймлайн, где пользователь может фиксировать свой культурный опыт (фильмы, книги, музыка, видеоигры) от детства до настоящего времени.

Какую проблему решаем: > Сейчас культурный опыт людей фрагментирован по разным платформам (Goodreads для книг, Letterboxd для кино, Spotify для музыки, трекеры для игр). Из-за этого невозможно увидеть цельную картину развития личности. TLOM ломает эти границы, объединяя все грани опыта в единый поток времени. Это не просто "список просмотренного", а визуализация пути.

Целевая аудитория: > Люди, увлекающиеся культурой, киноманы, геймеры, книголюбы и все, кто хочет сохранить свое цифровое и эмоциональное наследие, анализировать свои вкусы в ретроспективе и делиться этим с другими.

Ключевая механика (Core Loop):

Универсальные Карточки Контента: Любая запись (игра, фильм, песня) представляет собой интерактивную Карточку.

Отслеживание состояний: Карточка не просто добавляется в базу. Пользователь фиксирует события, привязанные к меткам времени (Timestamps): "Начал", "Поставил на паузу", "Бросил", "Завершил", "Оставил ревью/оценку".

Таймлайн (Хронология): Главный UI/UX элемент проекта — это визуальная лента времени, на которой эти карточки располагаются в зависимости от дат взаимодействия с ними.

Социальное взаимодействие: Возможность сделать свой таймлайн публичным, чтобы другие пользователи могли "посмотреть на мир глазами автора". Пользователи могут подписываться друг на друга (Follow), ставить лайки на публичные записи и оставлять комментарии. Лента подписок позволяет видеть активность людей, за которыми следишь.

Уведомления (Notifications): Система real-time уведомлений через SignalR. Пользователь получает мгновенные уведомления о событиях:
  — Социальные: «@maria подписалась на вас», «@alex лайкнул вашу запись», «@john оставил комментарий».
  — Системные: «Добро пожаловать в TLOM!», «Заполните свой профиль».
  — Для админа: «Пользователь @alex удалил запись #456», «Новый пользователь зарегистрирован».
Уведомления сохраняются в БД (персистентные) и одновременно отправляются через SignalR (real-time push).

Источники данных о контенте (External API Integration): Контент (фильмы, книги, игры, музыка) не вводится пользователем вручную. При добавлении карточки пользователь ищет контент, а система подтягивает данные из внешних API:
  — 🎬 TMDB (The Movie Database) — для фильмов.
  — 📚 Google Books API — для книг.
  — 🎮 Steam/CheapShark — для игр.
  — 🎵 Spotify API — для музыки.
Важный принцип: MediaItem — это общий пул. Первый пользователь, добавивший контент, инициирует его создание в БД. Все последующие пользователи переиспользуют уже существующую запись (поиск по ExternalId + ExternalSource).

Статистика и Аналитика пользователя: Платформа автоматически собирает и визуализирует статистику культурного опыта пользователя. Примеры:
  — Годовые отчёты в стиле Spotify Wrapped: «В 2025 ты прочитал 23 книги, прошёл 8 игр и посмотрел 47 фильмов».
  — Любимые жанры, средние оценки, самые продуктивные месяцы.
  — Общее время, потраченное на каждый тип контента.
  — Персональные рекорды и тренды вкусов по годам.

Лендинг (Landing Page — первое впечатление): При заходе на сайт пользователя встречает полноэкранная анимированная история, показывающая путь человека через медиа и искусство на каждом этапе жизни:
  Сцена 1 — Детство: ребёнок играет в простую игру, ему читают сказку, смотрит мультфильм на старом телевизоре, слушает музыку из телика.
  Сцена 2 — Юность (студент): играет в игры на ПК, слушает музыку в наушниках, читает книгу за столом, смотрит фильм.
  Сцена 3 — Взрослая жизнь: играет в зрелые игры, читает серьёзную литературу, слушает музыку в машине, ходит в кинотеатр.
  Сцена 4 — Зрелость: спокойные игры, классическая литература, музыка на виниле/радио, фильмы с семьёй.
После анимации — скролл вниз открывает демо-таймлайн: вертикальная лента от года рождения до текущего возраста, по которой скользят плавающие карточки фильмов, книг, игр и музыки. В конце — призыв к действию «Начни свою историю» с кнопками входа.
Технически: CSS/JS анимации (Framer Motion или GSAP + ScrollTrigger), scroll-driven animation, lazy-loading ресурсов, WebP/AVIF.

Локализация и Темы (UI/UX Features): 
Платформа поддерживает динамическую смену визуальных тем и языков:
  — Темы (Theming): Поддержка светлой и темной темы (Dark Mode) с автоматическим определением системных настроек.
  — Интернационализация (i18n): Полная поддержка мультиязычности (Азербайджанский, Русский, Английский). Пользователь может переключать язык "на лету" без перезагрузки страницы.

Аутентификация и Регистрация: Регистрация и вход через Google OAuth 2.0 (внешний провайдер). Пользователь может войти в один клик через свой Google-аккаунт. Также поддерживается классическая регистрация через email/пароль. JWT-токены для поддержания сессии.

Админ-панель и Аудит: Администратор имеет полный доступ к системе аудита. В БД фиксируются все действия пользователей: кто, когда и какую карточку создал, изменил или удалил. На сайте админ может просматривать полную информацию о карточках любого пользователя, редактировать или удалять их. Поддержка фильтрации и поиска по аудит-логам. Администратор получает real-time уведомления (SignalR) о ключевых действиях пользователей.

Авторизация и Защита доступа: Трёхуровневая система авторизации:
  — Аутентификация: JWT Middleware — все защищённые эндпоинты требуют валидный токен.
  — Ролевая авторизация: [Authorize(Roles = "Admin")] — админ-эндпоинты недоступны обычным юзерам.
  — Resource-based авторизация: пользователь может изменять/удалять только свои записи (Entry, Comment, Profile). Проверка через ResourceOwnerHandler (ASP.NET Core Authorization Policies).
